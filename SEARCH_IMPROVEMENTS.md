# بهبودهای سیستم جستجو

## تغییرات انجام شده

### 1. ایجاد صفحه نتایج جستجو (`src/pages/SearchResults.jsx`)

صفحه جدیدی برای نمایش نتایج جستجو ایجاد شد که شامل:

- **جستجوی زنده**: جستجو به صورت خودکار هنگام تغییر query در URL
- **مدیریت حالت‌ها**: Loading, Error, Empty states
- **فرم جستجو**: با قابلیت پاک کردن و جستجوی سریع
- **نمایش نتایج**: کارت‌های زیبا برای هر دارو
- **ناوبری**: لینک مستقیم به صفحه جزئیات هر دارو

### 2. بهبود کامپوننت HeroSection

- **ناوبری به نتایج**: جستجو به جای console.log، کاربر را به صفحه نتایج هدایت می‌کند
- **جستجوی سریع**: تگ‌های سریع اکنون به صورت خودکار جستجو را انجام می‌دهند
- **پشتیبانی از Enter**: امکان جستجو با کلید Enter
- **استفاده از useNavigate**: برای ناوبری بهتر

### 3. اضافه کردن Route جدید

Route جدید `/search` به `App.jsx` اضافه شد که از query parameters استفاده می‌کند.

## ویژگی‌های جدید

### ✅ جستجوی کامل
- جستجو از طریق API GapGPT
- نمایش نتایج به صورت کارت‌های زیبا
- پشتیبانی از جستجوی فارسی

### ✅ UX بهتر
- Loading states
- Error handling
- Empty states با پیام‌های مناسب
- پشتیبانی از Enter key
- دکمه پاک کردن در input

### ✅ ناوبری بهتر
- استفاده از URL parameters برای share کردن نتایج
- Back navigation
- لینک مستقیم به جزئیات داروها

### ✅ جستجوی سریع
- تگ‌های پیشنهادی
- جستجوی یک کلیکی
- حفظ query در URL

## نحوه استفاده

### از صفحه اصلی:
1. کاربر در input جستجو تایپ می‌کند
2. روی دکمه "جستجو" کلیک می‌کند یا Enter می‌زند
3. به صفحه `/search?q=عبارت_جستجو` هدایت می‌شود
4. نتایج به صورت خودکار نمایش داده می‌شوند

### از تگ‌های سریع:
1. کاربر روی یکی از تگ‌ها کلیک می‌کند
2. به صورت خودکار به صفحه نتایج هدایت می‌شود
3. جستجو انجام می‌شود

### از صفحه نتایج:
1. کاربر می‌تواند دوباره جستجو کند
2. می‌تواند query را تغییر دهد
3. می‌تواند روی هر دارو کلیک کند تا جزئیات را ببیند

## ساختار URL

```
/search?q=عبارت_جستجو
```

مثال:
```
/search?q=آموکسی‌سیلین
/search?q=مسکن
/search?q=سردرد
```

## بهبودهای آینده

1. **Debouncing**: برای کاهش درخواست‌های API
2. **Auto-complete**: پیشنهادات هنگام تایپ
3. **فیلترها**: فیلتر بر اساس دسته‌بندی، قیمت و ...
4. **مرتب‌سازی**: بر اساس مرتبط‌ترین، جدیدترین و ...
5. **Pagination**: برای نتایج زیاد
6. **جستجوی پیشرفته**: با چندین فیلتر
7. **تاریخچه جستجو**: ذخیره جستجوهای اخیر
8. **جستجوی صوتی**: برای دسترسی بهتر

## نکات فنی

- استفاده از `useSearchParams` برای مدیریت query parameters
- استفاده از `useEffect` برای جستجوی خودکار
- مدیریت state برای loading و error
- استفاده از `encodeURIComponent` برای امنیت URL
- پشتیبانی از RTL برای فارسی
